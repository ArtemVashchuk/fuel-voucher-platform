## 2. USER FLOW (MOBILE APP)

1. User opens app → sees **list of stations** (OKKO, WOG, UPG, KLO).
2. User selects station → sees **fuel types** for that station.
3. User selects fuel → sees **package sizes**:

   * 10 L
   * 20 L
   * 50 L
4. User selects package → sees:

   * Official station price
   * Discounted price
5. User clicks **Buy** → redirected to **Stripe Checkout**.
6. Payment succeeds → Stripe webhook triggers backend logic.
7. Backend:

   * Finds an **available QR code** matching:

     * Station
     * Fuel type
     * Liters
   * Assigns QR to purchase
   * Marks QR as **sold**
8. User navigates to **My Fuel Codes** → sees purchased QR code.
9. User **never sees inventory or QR counts**.

---

## 3. BACKEND BUSINESS LOGIC (CRITICAL)

### Purchase Flow

* User buys `FuelPackage(id = pkg_10L_OKKO_95)`

### Stripe Webhook

On successful payment:

* Query available QR where:

  * stationId
  * fuelTypeId
  * liters
  * status = "available"
* If found:

  * Assign QR to purchase
  * Set `Purchase.status = "delivered"`
  * Set `QrCode.status = "sold"`
  * Save QR image URL into purchase

### User Query

* Mobile app fetches:

  ```
  purchases
  where userId = currentUser
  and status = "delivered"
  ```
